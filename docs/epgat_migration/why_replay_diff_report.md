# Why Replay Diff Report

## 1. Executive Summary

- Celegans: no direct label-direction bug, no eval/export probability-column bug, and no merge-key misalignment were found; the most likely explanation is replay-fidelity mismatch, with the current replay run far shorter and less faithful than the old setup.
- Human: the current replay is not the same condition as the old ~0.91 figure. The old figure is derived from processed summary tables, while the current replay is a single low-epoch direct run.

## 2. Celegans Diagnosis

### Label direction check
- Old EPGAT celegans labels come from `EPGAT/data/essential_genes/celegans/EssentialGenes/ogee.csv` with `Label=1` used as essential.
- New replay keeps `Label -> label` without inversion in `src/data/epgat_legacy_label_adapter.py`.
- Conclusion: no label-direction mismatch found. Confidence: high.

### Probability direction check
- Current metrics use `pred_score = sigmoid(logits)` from `train_epgat_legacy.py`; this is the saved probability used by both evaluation and export.
- Test-split AUROC/AUPRC with current score: 0.2418 / 0.0589.
- Test-split AUROC/AUPRC with `1 - score`: 0.7582 / 0.2573.
- This shows anti-correlated test predictions, but not an evaluator/exporter direction bug because both use the same probability column and that column is explicitly `P(y=1)`.
- Conclusion: probability-direction implementation issue: no. Anti-correlation on the test fold: yes. Confidence: high.

### Merge / ID alignment check
- Merge key is `legacy_gene_id`.
- Builder merges features and labels explicitly by key, not by row order.
- No duplicate keys were found in `node_manifest.tsv` or `label_manifest.tsv`.
- No manifest key-set mismatch was found.
- Conclusion: no merge-alignment bug found in the audited artifacts. Confidence: high.

### Can this explain AUROC < 0.5?
- Yes, but only as anti-correlated held-out behavior under a low-fidelity replay run, not as a label/export/merge bug.
- The current replay uses only 5 epochs and a single seed, whereas the old legacy logs use 100 epochs and 5-run aggregation.
- Primary cause ranking for celegans:
  1. Replay-fidelity mismatch / under-matched old condition (medium confidence)
  2. Probability anti-correlation as a symptom on the test fold, not an eval bug (medium confidence)
  3. Label direction error (rejected; high confidence)
  4. Merge / ID alignment error (rejected; high confidence)

## 3. Human Diagnosis

### Old figure source
- The published/summary figure path is `utils/plots1.py` and `utils/plots2.py`.
- Those scripts read `outputs/results/human_final.csv`.
- `human_final.csv` is generated by `utils/prepare_data.py`, which aggregates multiple result directories (`First/Second/Third`) and applies additional post-processing, including forced GAT/GCN metric swaps when GAT is lower.

### Old experiment condition
- Raw old human GAT full-omics log: `logs/human_run_gat_exp_ort_sub.log`.
- Command: `python -m runners.run_gat --organism human --ppi string --train --seed 42 --n_runs 5 --sublocs --expression --orthologs`.
- Old runner uses 100 epochs.
- Old raw `GAT_results.csv` human full-omics row is `0.8905`, not ~0.91 exactly.
- The ~0.91 range aligns with threshold-sweep / processed rows in `results/GAT_results-string.csv`, especially `GAT_SUB_ORT` and some `GAT_EXP_SUB_ORT`/`GAT_EXP_ORT` rows.

### Current replay condition
- Config: `configs/epgat_legacy_human.yaml`.
- Single run only.
- Seed = 1029.
- Epochs = 2.
- Fixed threshold = 500.
- Feature blocks in replay schema: orthologs, expression, sublocalization, degree.
- Output metric is direct single-run test AUROC/AUPRC/accuracy/F1/MCC, not processed multi-directory summary.

### Most likely reasons for the gap
  1. Not the same condition as the old figure (high confidence)
  2. Training schedule / repeat strategy mismatch: 100 epochs and 5 runs vs 2 epochs and 1 run (high confidence)
  3. Old figure provenance mismatch: processed `human_final.csv` vs raw replay metrics (high confidence)
  4. Feature-contract mismatch (low confidence; degree and block order appear compatible)
  5. Label-set mismatch (low confidence; source and class counts match)
  6. A true code bug (low-to-medium confidence; not supported by current audit)
