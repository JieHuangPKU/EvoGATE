# Phase 2A Old-Result Audit

## Executive Decision

- The previous Phase 2A replay benchmark was based on the legacy raw runtime feature matrix and is now archived.
- The formal Phase 2A target has been replaced by old EPGAT original-result reproduction.
- For human, the operative target is the threshold-sweep result table, not the fixed replay matrix.

## Old Final Result Lineage

### A. Raw model result tables

Legacy result sources include:

- `EPGAT/results/GAT_results.csv`
- `EPGAT/results/GAT_results-string.csv`
- `EPGAT/results/GCN_results.csv`
- other baseline result tables consumed by the old plotting utilities

For the human `~0.91` neighborhood, the strongest direct match is in:

- `EPGAT/results/GAT_results-string.csv`

Relevant human rows there include:

- `GAT_SUB_ORT @ string_thr=100 -> AUROC 0.9117563769`
- `GAT_SUB_ORT @ string_thr=200 -> AUROC 0.9139347709`
- `GAT_SUB_ORT @ string_thr=300 -> AUROC 0.9141008214`
- `GAT_EXP_SUB_ORT @ string_thr=300 -> AUROC 0.9128556853`

### B. Post-hoc final CSV builder

Legacy `final.csv` files are generated by:

- `EPGAT/utils/prepare_data.py`

That script:

- reads result cohorts from hard-coded directories:
  - `outputs/results/First`
  - `outputs/results/Second`
  - `outputs/results/Third`
- loads per-model CSV files such as `GAT_results.csv`, `GCN_results.csv`, `MLP_results.csv`, `RandomForest_results.csv`
- averages metrics across available rows with the same `name` and `organism`
- then mutates the summary

### C. Non-scientific mutations applied by `prepare_data.py`

`prepare_data.py` does more than aggregate:

- it swaps GAT and GCN values whenever `GAT < GCN` for the same suffix
- it rescales standard deviations by heuristic rules
- it subtracts `0.05` from RF value metrics for `human`, `celegans`, and `coli`
- it writes `outputs/results/<organism>_final.csv`

Therefore:

- `human_final.csv` is a presentation-layer artifact
- it is not a clean scientific target by itself
- its lineage must be tracked, but it should not be used as the sole trusted ground truth for reproduction

### D. Plot consumers

Legacy plotting scripts read the post-hoc final files directly:

- `EPGAT/utils/plots1.py`
- `EPGAT/utils/plots2.py`

Those scripts consume:

- `outputs/results/human_final.csv`
- `outputs/results/celegans_final.csv`
- `outputs/results/yeast_final.csv`
- and the other species `_final.csv` files

## Phase 2A Operational Contract

### Primary target table

- Use `EPGAT/results/GAT_results-string.csv` as the formal target definition for human Phase 2A.
- Treat `human_final.csv` as downstream provenance that must be explained, not blindly copied.

### Supported feature subsets

The migrated builder now supports explicit subset control via config:

- `expression: true|false`
- `orthologs: true|false`
- `sublocalization: true|false`
- `include_degree: true|false`

This enables exact reproduction of:

- `EXP`
- `SUB`
- `ORT`
- `EXP_SUB`
- `EXP_ORT`
- `SUB_ORT`
- `EXP_SUB_ORT`

### Degree handling

- Degree is no longer enabled by default for Phase 2A.
- It must be explicitly requested in config with `include_degree: true`.
- The current formal human configs keep `include_degree: false`.

### Why this target

- It is the clearest human old-result point near the remembered `~0.91`
- It belongs to the old threshold-sweep table directly
- It avoids conflating the target with `prepare_data.py` post-processing
